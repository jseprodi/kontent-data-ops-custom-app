Help & Documentation

Overview

The Kontent.ai Data-Ops Custom App provides a user-friendly interface for running data-ops commands. This app serves as a graphical wrapper around the Kontent.ai data-ops command-line tool.

Key Features

  • Command Selection: Choose from all available data-ops commands
  • Environment Management: Select source and target environments
  • Real-time Output: View command execution output in real-time
  • Batch Operations: Run multiple commands in sequence or parallel
  • Workflows: Create reusable command sequences
  • History & Statistics: Track command execution history and statistics

Commands

Workflows

Workflows allow you to run multiple commands in sequence or parallel, making it easy to automate complex operations.

Creating a Workflow

  1. Click the "Workflows" button in the header
  2. Click "New Workflow" to create a new workflow
  3. Enter a name for your workflow
  4. Add steps by clicking "+ Add Step"
  5. Configure each step with a command and its options
  6. Choose execution mode: Sequential or Parallel
  7. Save the workflow

Running a Workflow

Click "Run Workflow" to execute all steps. If "Stop on error" is enabled, the workflow will stop if any step fails.

Workflow Templates

Use pre-built templates to quickly create common workflows like "Backup ‚Üí Sync" or "Backup ‚Üí Restore".

Troubleshooting

Common Issues

Command Not Found

If you see "Command not found" errors:

  • Ensure the data-ops CLI is installed: npm install -g @kontent-ai/data-ops
  • Check that the CLI is in your PATH
  • Verify the server can access the CLI

API Key Errors

If you encounter API key authentication errors:

  • Verify your Management API key is correct
  • Check that the API key has the required permissions
  • Ensure the environment ID matches the API key

Connection Issues

If commands fail to connect:

  • Check your internet connection
  • Verify the Kontent.ai service is accessible
  • Check firewall settings

Frequently Asked Questions

Q: Can I run multiple commands at once?

A: Yes! Use the Workflows feature to create sequences of commands that run automatically.

Q: How do I save my command configurations?

A: Use Command Templates to save frequently used configurations. You can also save Environment Profiles for quick environment switching.

Q: Can I undo a command?

A: Some commands support rollback. Check the command documentation for details. Always test in non-production environments first.

Q: How long is command history kept?

A: By default, up to 100 command executions are kept. You can adjust this in Settings.

Q: Can I export my data?

A: Yes! You can export command history, templates, profiles, and all app data from the Settings panel.
